# Logs

Telemetry, session records, and CI pipeline run logs.

## telemetry/

JSONL files. Each line is one event. Schema:

```jsonl
{"ts":"<ISO8601>","event":"<name>","agent":"<role>","task":"<slug>",...}
```

Common events: `session_start`, `session_end`, `test_run`, `commit`, `deploy`, `error`.

Files are named `YYYY-MM-DD.jsonl`. Rotate daily.

## sessions/

Per-session markdown logs. Named `YYYY-MM-DD-<slug>.md`. Contains a free-form narrative of what the agent did, decisions made, and what to hand off to the next session.

## pipeline/

CI/CD run records generated by `.github/workflows/`. Named by run ID. Keeps the last 50 runs; older ones are pruned by the pipeline itself.
